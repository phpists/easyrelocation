Package ID vs. Keyword installs: difference, importance and relevance

Many novice developers often make the same mistake: they start the invalid promotion. What are we talking about? How can paid promotion be wrong? What to do and why is this happening?

Well, first of all it is necessary to clarify that there is no wrong promotion, but there is a wrong choice of promotion. We will explain this further, but first we will explain what types of promotion we offer, what is the difference between them and why it is so important to make the right choice.
So, we offer two types of installs:

  • Package ID installs (or direct link installs)
  • Keyword installs
Let's consider both types of promotion in more detail.
What does «package ID installs» mean?
Package ID - is a unique name that identifies your app. Simply put, package install is a direct install by a package id and clients usually order package installs if they just want to have more installs in the statistics. Let's see what kind of advantages and disadvantages have package ID installs.
Pros:

  • It is much cheaper than keywords installs;
  • It is a good option to promote your app from scratch, if you don't have any installs at all;
  • Package id install will help your app start ranking for keywords faster.
    Cons:

    • It is not effective if your app already has enough installs;
    • Package ID installs don't boost keyword ranks. So they won't lead to more organics;
    • It can take some time to see the result.
      What does «keywords installs» mean?
      Keyword install - is an install by keyword (by which a user searches the app). Also, keyword installs are more effective as they increase the rank. So, keyword installs are important to boost your apps' rank and get a lot of organic installs. The keywords are words or phrases that represent search queries for which the app is searched by users. So, let's see on pros and cons of keyword installs:
      Pros:

      • Keywords are more effective for apps, which already have enough installs;
      • Campaign for keywords installs is a good way to combine a lot of different particular keywords and run it at the same time;
      • Keywords promotion is 100% effective and after boosting with keyword installs your app will get higher in ranks, more visibility and more chances for users to find and download your app.
              Cons:

              • It is not effective for new apps because usually they don't have keywords;
              • If you don't have your keywords, it will be difficult to find the best one (but you can use ASO services to solve this little problem);
              • Sometimes you can drop out of the keyword rank because of the changes in google algorithm.
                  Returning to the topic of our article, we want to emphasize that only by understanding the difference between these types of advancement can a good result be achieved. It is normal if you did not know what exactly would be effective for you. You can always understand the difference between installs using the services of Keapp.top and the advices of the support department.
                  As a conclusion:
                  Package install is a direct install by a package id, keyword install is an install by keyword (by which a user searches the app). Keyword installs are more effective as they increase the rank, clients order package installs if they just want to have more installs in the statistics or start ranking for keywords if their app is new. We really recommend keyword installs to boost your apps' rank!
